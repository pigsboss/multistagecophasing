# 🌙 LUNAR-SWING: 地月共振摆动轨道与大气穿越维持技术研究

## 🎯 项目概述

**LUNAR-SWING** 是 MCPC 框架下的首个专项研究项目，旨在通过探索地月三体系统中的共振摆动轨道，**同时驱动并验证框架的核心能力升级**。本项目具有双重目标：

1.  **专项轨道研究**：探索具有大气穿越和月球引力维持特性的共振摆动轨道。
2.  **框架能力夯实**：以高精度轨道仿真需求为牵引，填补框架在**高精度动力学环境建模**和**数值算法**方面的关键空白。

该轨道具有以下特征：
- **近地点**：低于 200 km（LEO 轨道高度），穿越地球大气层上部
- **远地点**：接近月球轨道，利用月球引力进行轨道能量调节
- **轨道维持**：通过月球引力补偿大气阻力损耗，实现长期轨道稳定性
- **共振特性**：与月球轨道形成稳定共振比（如 2:1、3:2 等）

## 🔬 科学目标

1.  **轨道设计验证**：证明地月共振摆动轨道的存在性与稳定性。
2.  **维持机制验证**：验证"大气穿越-月球引力补偿"轨道维持新概念。
3.  **框架能力拓展**：
    - **环境模型**：集成高精度太阳系星历（如 DE440）、高阶地球重力场模型（如 EGM2008）。
    - **数学工具**：引入高精度数值积分器（如 RKF78）、插值算法和线性代数求解库。
    - **验证体系**：建立模型与标准软件（如 GMAT、STK）的比对验证流程。
4.  **算法基础建立**：为后续 L3 级日地三体轨道搜索奠定数值算法（如微分修正、打靶法）基础。

## 🔬 架构定位：MCPC 数字镜像中的一个验证场景

**lunar-swing** 是在 MCPC 构建的**数字镜像世界**中进行的一个特定验证场景。

### MCPC：真实航天器系统的数字平行世界
MCPC 是一个分层数字框架，如同《盗梦空间》中的梦境层级：

**第0层：现实世界**
- 你同时设计的真实空间分布式光学干涉探测系统
- 真实航天器、真实空间环境、真实物理定律

**第1层：MCPC 数字镜像**（本框架）
- 建立真实世界的完整数字映射
- 用于在数字域验证、测试、优化真实世界的设计

其中，MCPC 数字镜像内部进一步分层：
- **核心三域（core）**：定义数字世界的基本法则
  - 时空域：数字世界的时空基准（坐标系、历元、星历）
  - 物理域：数字世界的物理规律（引力、阻力、光压等）
  - 信息域：数字世界的信息过程（感知、通信、计算、控制）
  
- **幕后两域**：操作和观察数字实验
  - 编排域（simulation）：调度数字实验的流程
  - 分析域（analysis）：解读数字实验的结果

### lunar-swing：数字世界中的一个实验场景
在 MCPC 数字镜像中，我们设置这样一个实验：

```
实验名称：地月共振摆动轨道数字验证
实验层级：MCPC 数字镜像（第1层）
映射关系：
  现实世界概念          →    MCPC数字镜像配置
  ──────────────────────────────────────────
  真实航天器            →    球对称模型（cannonball）
  真实轨道环境          →    三体引力+大气阻力模型  
  真实导航控制          →    理想导航+理想控制模型
  真实任务概念          →    simulation编排的实验流程
  真实数据分析          →    analysis域的效能评估
```

### 实验目标：
1. **数字验证**：在数字镜像中验证地月共振摆动轨道的数学可行性
2. **框架检验**：测试 MCPC 数字镜像的以下能力：
   - 时空域：地月旋转坐标系的正确实现
   - 物理域：三体引力与大气阻力的耦合仿真
   - 信息域：轨道维持控制策略的数字验证
3. **能力反哺**：通过实验需求，推动数字镜像本身的能力升级
   - 增加高精度星历支持
   - 完善高阶重力场模型
   - 优化数值积分算法

### 重要区分：
- **这不是真实任务**：而是真实任务在数字世界的"预演"
- **这不产生真实数据**：但产生的数字数据将指导真实任务设计
- **这不消耗真实燃料**：但计算的数字燃料消耗将影响真实任务规划

通过这个相对简化的数字实验，我们既验证了一个有趣的轨道概念，也**压力测试了 MCPC 数字镜像本身的可靠性和完备性**。

## 🏗️ 技术架构

本项目在 MCPC 现有架构基础上进行以下拓展：

### 1. 通用三体引力模型
- 重构硬编码的 CRTBP 实现为通用基类
- 支持任意双主天体系统配置
- 提供地月、日地等预设系统

### 2. 增强摄动模型库
- 地球非球形引力场（J2-J6）
- 地球大气阻力模型（指数模型）
- 月球外大气层模型
- 太阳光压（为日地系统准备）

### 3. 轨道设计算法
- 共振轨道搜索算法（打靶法）
- 摆动轨道优化器
- 稳定性分析工具

### 4. 专用仿真场景
- 地月摆动轨道长期演化仿真
- 大气穿越过程高精度模拟
- 燃料消耗与轨道维持策略评估

### 5. 框架基础设施升级（核心能力夯实）

本项目将系统性地填补 MCPC 框架在高精度仿真方面的关键空白：

#### 5.1 高精度星历系统 (`HighPrecisionEphemeris`)
- **目标**：支持 JPL DE440/DE430 等标准星历格式
- **功能**：
  - 提供太阳、地球、月球等天体的精确位置/速度
  - 支持多种时间系统转换（TT, TDB, UTC）
  - 实现高效插值算法（拉格朗日、切比雪夫）
- **集成路径**：扩展 `mission_sim/core/spacetime/ephemeris/` 模块

#### 5.2 高阶地球重力场模型 (`HighOrderGeopotential`)
- **目标**：支持 EGM2008 等地球重力场模型，阶次可配置（最高 360x360）
- **功能**：
  - 实现田谐项和扇谐项加速度计算
  - 支持球谐系数文件读取（ICGEM 格式）
  - 提供梯度计算用于高精度轨道外推
- **集成路径**：扩展 `mission_sim/core/physics/models/` 模块

#### 5.3 增强数值算法库
- **目标**：提供工业级数值计算工具
- **功能**：
  - 变步长积分器（RKF78, DOP853）
  - 稀疏矩阵运算优化
  - 微分修正和打靶法算法优化
- **集成路径**：扩展 `mission_sim/utils/` 模块

#### 5.4 验证与比对系统
- **目标**：建立与行业标准软件的比对流程
- **功能**：
  - 与 GMAT/STK 仿真结果的自动化误差分析
  - 蒙特卡洛打靶的统计验证
  - 模型精度评估报告生成

## 📊 数字实验实施路线（Sprint 驱动）

在 MCPC 数字镜像中实施本实验场景，我们采用敏捷开发模式，共分为 **四个阶段（8个Sprint）**。

**详细实施框架请参阅：** [00. LUNAR-SWING 项目概览与分阶段实施框架](00_project_overview.md)

### 阶段概览
1. **第一阶段：概念设计与基础设施准备** (Sprint 1-2)
   - 完成顶层设计与框架能力缺口分析
   - 定义新模块接口契约，建立测试框架

2. **第二阶段：核心算法原型实现** (Sprint 3-4)
   - 实现通用三体动力学与共振轨道搜索引擎
   - 在简化模型下验证轨道存在性

3. **第三阶段：高保真数字实验** (Sprint 5-6)
   - 集成高精度摄动模型，执行长期演化仿真
   - 验证轨道在真实摄动下的稳定性

4. **第四阶段：框架集成与能力反哺** (Sprint 7-8)
   - 将验证组件标准化并入MCPC核心库
   - 完成技术转移与知识沉淀

**核心原则**：每个功能都有测试，每个阶段都有可视化产出。
